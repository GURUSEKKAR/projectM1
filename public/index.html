CI/CD Security Pipeline Flow

This interactive diagram shows an end-to-end GitHub Actions pipeline with build, security scans, deployment, and notifications.

Step 1: CODE COMMIT

  • Developer pushes code to main branch
▼

Step 2: TRIGGER

  • GitHub webhook triggers Actions workflow
▼

Step 3: BUILD

  • Checkout code
  • docker build -t myapp:latest .
  • Time: ~2–3 minutes
▼

Step 4: SECURITY SCAN #1 – Trivy

  • trivy image --severity HIGH,CRITICAL myapp:latest
  • Decision point: CRITICAL found?
YES ✖ Stop pipeline (exit 1)
NO ✔ Continue
  • Time: ~1–2 minutes
▼

Step 5: SECURITY SCAN #2 – OWASP Dependency-Check

  • dependency-check --scan ./
  • Generate JSON + HTML reports
  • Upload as artifact
  • Time: ~2–3 minutes
▼

Step 6: SECURITY SCAN #3 – SonarQube

  • sonar-scanner
  • Push results to SonarCloud
  • Quality Gate check (optional blocking)
  • Time: ~1–2 minutes
▼

Step 7: PUBLISH

  • docker tag myapp:latest username/myapp:latest
  • docker push username/myapp:latest
  • Time: ~1–2 minutes
▼

Step 8: DEPLOY TO EC2

  • SSH to EC2
  • docker pull username/myapp:latest
  • Stop old container (if exists)
  • docker run -d -p 80:3009 myapp:latest
  • Time: ~1–2 minutes
▼

Step 9: SECURITY SCAN #4 – OWASP ZAP

  • zap-baseline.py -t http://<ec2-ip>
  • Attack live application
  • Generate HTML report
  • Upload as artifact
  • Time: ~3–5 minutes
▼

Step 10: UNIFIED REPORTING

  • Aggregate all JSON reports
  • Generate single HTML dashboard
  • Upload as artifact
  • Time: ~30 seconds
▼

Step 11: NOTIFICATION

  • GitHub Actions shows: ✔ Success or ✖ Failed
  • Developer gets email (if configured)
  • Total pipeline time: ~15–20 minutes

Built with ❤️ using Node.js & Express.
